Навигация

В модуле Офисе в режиме development все классы доступны как есть (при наличии разрешений). Однако удобнее иметь определенное иерархическое меню доступа к классам, отвечающее функциональности приложения. В нем можно задать порядок расположения классов, группировку, отображение по требованию или фильтрацию объектов по различным критериям.

Навигация - это отдельная модель метаданных, использующаяся для создания удобного пользовательского меню.

Данные навигации состоят из секций и пунктов. Секция - это набор пунктов меню, располагаемых в определенном порядке. Например, секция main отвечает за формирование бокового меню в модуле Офис. Пункт - это ссылка на страницу сайта. Каждый пункт может иметь дочерние пункты, формируя древовидную иерархию меню.

Перейдите в Студию и выберите в боковом меню Секции навигации. Создайте секцию main. Создайте пункт articles (Статьи). Это будет родительский пункт для списков статей в различных статусах. Перейдите на вкладку Дочерние пункты и создайте пункт publicArticles (Публичные статьи). В поле Класс выберите Статья, а в поле Представление выберите Публичный список.

Кодовое имя пункта должно быть уникальным внутри секции.

Создайте еще один дочерний пункт allArticles (Все статьи), в котором укажите класс без представления, чтобы пользователь мог получить доступ ко всем статьям, в любом статусе.

Экспортируйте метаданные и перейдите в Офис. В боковом меню появятся пункты главной секции навигации.
