Класс Статья

Выберите в боковом меню пункт Классы. Здесь описываются базовые сущности приложения.

Создайте класс статьи, указав кодовое имя article. После сохранения класса на форме появятся новые вкладки и поля.

Кодовое имя каждого элемента данных (класса, атрибута, представления и других) является идентфикатором внутри приложения и должно быть уникально в своем контексте. Например, имя атрибута уникально внутри класса.

Кодовое имя может состоять только из символов a-z, A-Z, 0-9, дефиса и подчеркивания. Используйте единый стиль написания имен (рекомендуется lowerСamelСase, когда все слова пишутся слитно и с заглавной буквы кроме первого).

Выберите вкладку Атрибуты. При наведении курсора на таблицу атрибутов появится панель с доступными командами.

Создайте атрибут title (Заголовок). Кодовое имя атрибута подчиняется тем же правилам, что и имя класса. Кроме того некоторые служебные атрибуты могут начинаться с подчеркивания.

При создании атрибута необходимо указать его тип, который определяется сохраняемыми данными. Для заголовка укажите тип Строка. Установите флажки Обязательный и Уникальный. Они определяют правила валидации атрибута. Статья всегда должна иметь заголовок, который не должен повторяться в других статьях.

Перейдите на вкладку Поиск и установите флажки Общий поисковый и Поисковый в списке выбора. Теперь статьи можно будет найти поиском среди заголовков.

Перейдите на форму класса Статьи. Укажите значение ".title" в поле Шаблон заголовка. Шаблон определяет представление объекта в строковом виде. Например, для выпадающих списков и подобных. Если шаблон не указан, то выводится идентификатор объекта. Значение ".title" будет выбирать данные из атрибута Заголовок (подробнее см. Header).

Создайте атрибут content (Содержимое) с типом Текст. Он будет хранить текст статьи. Тип Текст отличается от строки увеличенным полем ввода и способом хранения в некоторых базах данных.

Создайте атрибут date (Дата) с типом Дата. Он будет хранить дату статьи.

Создайте атрибут status (Статус) с типом Строка. Он будет хранить состояние статьи. Статус может иметь только фиксированные значения. Перейдите на вкладку Перечисления и создайте новое перечисление. На форме перечисления создайте Элементы статусов:

  • draft - черновик
  • published - опубликовно
  • archived - архивировано

Сохраните все изменения и экспортируйте метаданные в папку приложения meta/app.
